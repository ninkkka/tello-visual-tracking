# Разработка системы управления летательным аппаратом на основе визуальных данных
ВКР бакалавра, СПбГЭТУ «ЛЭТИ», 2026.  
Направление 27.03.04 «Управление в технических системах».

## Логика работы
Квадрокоптер DJI Tello автономно следует за перемещаемым ArUco‑маркером, используя только камеру. ORB‑SLAM2 работает в фоновом режиме для начальной инициализации. Управление — трёхканальный ПИД‑регулятор (LR, UD, FB).

## Файлы
| Файл | Что делает |
|---|---|
| `final_flight_v17.py` | Основной скрипт: подключение к дрону, детектирование ArUco‑маркеров, расчёт ошибок, ПИД‑регулятор, отправка команд. Внутри - метод `stream_to_slam()`, передающий кадры в ORB‑SLAM2 по TCP :9999 |
| `slam_network.cpp` | C++ программа в Docker‑контейнере: принимает кадры с TCP :9999, передаёт в `SLAM.TrackMonocular()`, отправляет матрицу позы $T_{cw}$ обратно на TCP :9997 |
| `pid_identification.m` | MATLAB: чтение CSV‑логов ступенчатых тестов, усреднение, подбор моделей P1 и tf(2,0), расчёт fit‑процентов |
| `pid_tuning.m` | MATLAB: синтез коэффициентов ПИД‑регулятора через PID Tuner, нормировка под Tello SDK |
| `pid_validation.m` | MATLAB: сравнение теоретической реакции замкнутой системы с реальными переходными процессами из логов полёта |
